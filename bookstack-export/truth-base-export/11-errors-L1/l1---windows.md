Window Errors & Omissions
A window error can be misplacement, misinterpretation, incorrectly sized or missing.
Windows must be:
●

Sized to the opening it exists in (inside of frame to inside of frame).

●

Sized based on the laser data if the visual data does not match.

●

Divided if there is drywall in between window openings.

●

Drawn without divisions if windows are separated by mullions.
○

Breaks in the windows due to mullions are only drawn this way if the operator
requests it.

●

For ADS files (iGUIDE tasks with a black insight tag) window types need to be applied to
match the visual data
○

Windows for ADS files are typically similar sizing and will not exhibit much
variation. Aim to demonstrate consistency with similar windows. Ex. Rare to see
24”, 23” and 25” windows in the same construction, they’re likely all 24” and the
drafters sizing is inaccurate.

Please note for windows:
●

Ensure window length is as accurate as possible using the two yellow dashed lines in
the Pano Preview.

●

If the window edge is not visible or hidden behind a curtain use your best judgment to
estimate the remaining unseen length.

●

We will ignore roof windows, roof lights or skylights as we draft the exterior wall to
where it meets with the floor so those windows (if put in) would be misrepresented.

Window Error Tolerances
Windows have an error tolerance of 5cm/2". This means that if you have to adjust the size of
the window by 5cm/2", you do not have to attribute it as an error towards the Drafter. Simply
correct the window and move on. If you have to adjust the window by anything greater than
5cm/2" then you must attribute a level 1 error towards the Drafter. Note down the error on the
notepad in Draft and increase the tally by 1 per instance.
For iGUIDEs relating to ADS tasks, windows are required to be sized to 1” increments. Any
fractional sizing for windows on a task that has a black tag insight will result in an L1 error.
1. Good Example Notes:
a. L1 - Missing window near scan 002
b. L1 - Window oversized near scan 0001
c. L1 - Window undersized near 0003. 3’ from 2’5”
d. L1 - Window incorrectly drafted on curved wall near 020
2. Bad Example Notes:
a. Wrong window.
b. Window is the wrong size.
c. Window is curved.
When noting the error, it is imperative to include the appropriate error level, a relevant scan,
and a brief, but succinct description of the error. It is also important to note what was
changed, such as Good Example Notes c. The “Bad Example” provides no details for the Drafter
to find or reference their error. Providing “Bad Example” notes will not be tolerated.
*Note: The intent of the error tolerance is to ensure we are not overloading every Drafter with
several miniscule errors. This does not mean we do not correct the error in question. We
always correct everything whether it is within or outside of the error tolerance. The error
tolerance in this document is to be strictly adhered to, making up your own standards or
tolerances will not be permitted.
