Structural Error Tolerances
Structural objects will have an error tolerance of 5cm/2” in terms of size (Width or Length). If
you have to adjust the size of the object by anything greater than 5cm/2” then you must
attribute a level 1 error towards the Drafter. Note down the error on the notepad in Draft and
increase the tally by 1 per instance.
Structural object placement can be tricky so there will be an error tolerance of 5cm/2” in
regards to location. If you have to move a structural object greater than 2” into the correct
location then you must attribute a level 1 error towards the Drafter. Note down the error on
the notepad in Draft and increase the tally by 1 per instance.
There is no tolerance for missing objects or countertops. If there is any instance of a missing
object or countertop section then you must attribute a level 1 error towards the Drafter. Note
down the error on the notepad in Draft and increase the tally by 1 per instance. In properties
where there is a grid of structural objects, often found in commercial warehouses, if all of the
objects are missed, an L3 can be attributed for failure to follow standards. This is to avoid
overloading a drafter with L1 errors but still alert them to the mistake.
1. Good Example Notes:
a. L1 – incorrectly sized O1 object at 0003
b. L1 – O1 from O3 at scan 0010
2. Bad Example Notes:
a. Missed column.
b. Missing object.
c. Wrong object type.
Take a look at how much detail is provided for the “Good Example”. The location of the
problem/issue should be explained using the floor it is on, the pano/scans that can see it and
what room it’s contained in or next to. We must be descriptive by describing the issue and if
applicable, provide a solution or tips as to how to avoid it in future. The “Bad Example”
provides no details for the Drafter and is completely useless. Providing “Bad Example” notes
will not be tolerated.

*Note: The intent of the error tolerance is to ensure we are not overloading every Drafter with
several minuscule errors. This does not mean we do not correct the error in question. We
always correct everything whether it is within or outside of the error tolerance. The error
