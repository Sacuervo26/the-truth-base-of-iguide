Creating a New Building
This error involves failing to create a new building for a verifiable detached structure. It is
important that a new building be created for proper area reporting of the two (or three or
four) spaces.
Typically, operators will submit a garage on its own floor instead of its own building. If you can
determine that it is actually detached then a new building must be created. It is important that
the building can be proven to be detached.
While not a reason for an L3 error, QA should ensure the new building and floor created is
descriptive. Changing the label from generic “New Building” to “Detached Garage” is favourable
and shows an attention to detail that is important to our clients. Furthermore, the generic
“New Floor” label that is used for a new floor in a new building, can be changed to “Main” to be
more descriptive.

Creating New Buildings Error Tolerances
The only tolerance is if the Drafter cannot confirm if the structure is truly detached or
attached. If you can prove that the structure should be detached, then you must create a new
building for it and then you must attribute a level 3 error towards the Drafter. Note down the
error on the notepad in Draft and increase the tally by 1 per instance.
1. Good Example Notes:
a. L3 – Detached garage as seen from living room at scan 006
b. L3 – Insufficient data to prove garage is detached. Leave as separate floor.
2. Bad Example Notes:
a. Detach the second building.
b. Garage is detached.
c. Shed is detached.
Take a look at how much detail is provided for the “Good Example”. The location of the
problem/issue should be explained using the floor it is on, the pano/scans that can see it and
what room it’s contained in or next to. We must be descriptive by describing the issue and if
applicable, provide a solution or tips as to how to avoid it in future. The “Bad Example”
provides no details for the Drafter and is completely useless. Providing “Bad Example” notes
will not be tolerated.
*Note: The intent of the error tolerance is to ensure we are not overloading every Drafter with
several minuscule errors. This does not mean we do not correct the error in question. We
always correct everything whether it is within or outside of the error tolerance. The error
tolerance in this document is to be strictly adhered to, making up your own standards or
tolerances will not be permitted.
