Draft Error Correction
A Drafting Error Correction is a task or work order created by an operator to notify us
of a potential mistake on the iGUIDE Floor Plan. These errors can range from simple
cosmetic corrections such as a missing window or door, to major redrafting
corrections due to misalignment. The most common error corrections that get
submitted by operators are missing windows, missing doors, missing columns and
missing premium objects.
Draft Error Corrections unfortunately use Full/Original Data only. Light Data is not and
cannot be used to perform a Draft Update.
*This is on R&D’s Roadmap to allow us to use Light Data.

Draft Error Correction Procedure
The procedure at a glance is to download the published iGUIDE, fix the reported error
in question, QA the entire property to ensure nothing else is incorrect, then reupload
the iGUIDE back to the Portal.
1) While in the QA Queue, select the blue “claim next work order” button. Draft
Error Corrections will appear at the very top of the QA Queue.

Figure #2 - Claim Next Work Order

2) The Task Details Page will open with the Quality Technician as the claimer listed
under the QA section. No name will be listed under the Drafter section as there
is no Draft to QA workflow for updates.
a) The Description field will indicate what is involved with the Draft Error
Correction.
b) The Description field for the following example indicates that the operator
would like us to fix several errors related to the uploaded .jpg images.
*Note: Some descriptions are very thorough, others are not so some more
investigation may be required.
3) Review any images provided in the “Supporting Documents” section to help
better understand the request. Some requests may or may not have Supporting
Documents.

6) Once all errors caught by the operator have been corrected, perform a quick QA
check on the rest of the property to ensure nothing else was missed.
7) Click the verify button to ensure there are no errors. Mitigate any pending
warnings if applicable.
8) Once the Quality Assurance check is complete:
a) If no additional errors were discovered, proceed to save and export the
file.
b) If additional errors were discovered and fixed, note down what those
errors were in the Project Comments in Draft. Once noted, proceed to
save, export and close the file.
9) Scroll down to the comments section to describe what was done for the update.
For any update task it is best practice to leave a comment in the Comment
section of the Task Details Page to confirm what changes were made. This
serves as a changelog (revision history) and gives Planitar better traceability of
events.

11) After dropping or selecting the “draft.tar", ensure that the checkbox “Publish to
iGUIDE upon upload” is checked and the “Send iGUIDE Report” checkbox is
unchecked.
a) We want to publish the Update.
b) We do not want to resend the iGUIDE Report.
12) Finally, select the large blue “Upload” button and wait for the update to publish
to the Portal.
